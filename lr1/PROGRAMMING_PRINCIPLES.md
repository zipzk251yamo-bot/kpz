# Принципи програмування у проєкті

У цьому проєкті реалізовано низку фундаментальних принципів програмування, які забезпечують чистоту, перевикористанність та підтримку коду.

---

## 1. DRY (Don't Repeat Yourself — Не повторюйся)
Цей принцип спрямований на уникнення дублювання логіки. Замість написання однакових циклів для виводу масивів або пошуку елементів у різних частинах програми, логіка винесена в універсальні методи розширення.

* **Приклад у коді**: Метод `Print<T>` дозволяє виводити будь-який тип масиву (int, string тощо) без дублювання коду виводу.
* **Посилання**: [Extension.cs#L61-L71](Extension.cs#L61-L71)

## 2. SRP (Single Responsibility Principle — Принцип єдиної відповідальності)
Кожен клас у проєкті відповідає за одну конкретну задачу.

* **Приклад у коді**: Клас `ExtendedDictionaryElement` відповідає лише за структуру збереження даних (ключ та два значення). Клас `ExtendedDictionary` відповідає виключно за керування колекцією (додавання, видалення, пошук).
* **Посилання**: 
    * [ExpandedDictionary.cs#L11-L28](ExpandedDictionary.cs#L11-L28) (Клас елемента)
    * [ExpandedDictionary.cs#L31-L118](ExpandedDictionary.cs#L31-L118) (Клас словника)

## 3. KISS (Keep It Simple, Stupid — Роби це простіше)
Принцип підкреслює необхідність тримати код максимально простим і зрозумілим.

* **Приклад у коді**: Реалізація методу `WordCount` використовує простий і зрозумілий цикл `for` для підрахунку символів, що робить код легким для читання та налагодження.
* **Посилання**: [Extension.cs#L14-L23](Extension.cs#L14-L23)

## 4. Encapsulation (Інкапсуляція)
Приховання внутрішньої реалізації даних і надання доступу до них лише через чітко визначені інтерфейси (властивості та методи).

* **Приклад у коді**: Список `_items` у класі `ExtendedDictionary` є приватним (`private`), що захищає цілісність даних від прямого втручання зовні. Взаємодія з даними відбувається через публічні методи `Add` та `Remove`.
* **Посилання**: [ExpandedDictionary.cs#L35](ExpandedDictionary.cs#L35)

## 5. Використання Generic-типів (Generics)
Це дозволяє створювати структури даних та методи, які працюють з будь-яким типом, забезпечуючи при цьому строгу типізацію та безпеку під час компіляції.

* **Приклад у коді**: Клас `ExtendedDictionary<T, U, V>` та методи розширення масивів `Search<T>` працюють з будь-якими типами даних.
* **Посилання**: 
    * [Extension.cs#L42](Extension.cs#L42)
    * [ExpandedDictionary.cs#L31](ExpandedDictionary.cs#L31)

## 6. Extension Methods (Методи розширення)
Цей механізм дозволяє "додавати" нові методи до існуючих типів без зміни їхнього початкового вихідного коду.

* **Приклад у коді**: Створення методів `WordCount` та `Invert` для стандартного типу `string`, а також методів для масивів.
* **Посилання**: [Extension.cs#L11-L40](Extension.cs#L11-L40)
